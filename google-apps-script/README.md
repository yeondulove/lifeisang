# 채용 지원서 → 구글시트 연동 설치 가이드

홈페이지 채용 지원 폼(`/` 홈의 "채용" 섹션)을 제출하면, 이 스크립트를 통해
**구글시트에 자동으로 한 줄씩 쌓이고**, 첨부파일(이력서·경력기술서·포트폴리오)은
**구글드라이브 폴더에 저장되어 시트에 링크로 표시**됩니다.

배포는 회원님의 구글 계정으로 딱 한 번만 하면 되고, 이후에는 자동으로 동작합니다.

## 1단계 — 구글시트 만들기

1. [sheets.google.com](https://sheets.google.com) 에서 새 스프레드시트 생성
2. 이름을 예: **"라이프해킹 채용 지원자"** 로 지정
3. 시트는 비워둬도 됩니다 (스크립트가 첫 실행 시 헤더를 자동 생성합니다)

## 2단계 — Apps Script 붙여넣기

1. 방금 만든 시트 화면에서 상단 메뉴 **확장 프로그램 → Apps Script** 클릭
2. 기본으로 열려 있는 `Code.gs`의 내용을 전부 지우고, 이 폴더의
   [`Code.gs`](./Code.gs) 파일 내용을 **전체 복사해서 붙여넣기**
3. 왼쪽 위 프로젝트 이름을 "채용 지원서 수집"처럼 바꿔주면 알아보기 쉽습니다
4. **저장** (Ctrl+S 또는 저장 아이콘)

## 3단계 — 웹 앱으로 배포

1. 오른쪽 위 **"배포" → "새 배포"** 클릭
2. 유형 선택(톱니바퀴 아이콘) → **"웹 앱"** 선택
3. 설정:
   - **설명**: 아무거나 (예: v1)
   - **다음 사용자 인증 정보로 실행**: **나(본인 계정)**
   - **액세스 권한이 있는 사용자**: **모든 사용자** ⚠️ 반드시 이 옵션이어야 홈페이지에서 접속 가능합니다
4. **배포** 클릭
5. **권한 승인 요청**이 뜨면:
   - 본인 구글 계정 선택
   - "이 앱은 Google에서 확인하지 않았습니다" 경고가 뜨면 **고급 → (프로젝트명)으로 이동(안전하지 않음)** 클릭 → **허용**
   - (본인이 만든 스크립트라 안전합니다. 외부에 공개 배포하는 게 아니라 본인 전용 자동화 스크립트라서 나오는 표준 경고입니다.)
6. 배포가 완료되면 **"웹 앱 URL"**이 표시됩니다. 이 URL을 복사하세요.
   - 형태 예시: `https://script.google.com/macros/s/AKfycb.../exec`

## 4단계 — 웹사이트에 URL 연결

복사한 웹 앱 URL을 저에게 알려주시면, 제가 코드에 붙여넣고 배포하겠습니다.

(또는 직접 하시려면: `lifehacking-landing/components/about/AboutContent.tsx` 상단의
`const APPS_SCRIPT_URL = "여기에_붙여넣기";` 부분에 넣고 커밋·푸시하면 됩니다.)

## 5단계 — 테스트

홈페이지 채용 섹션에서 테스트로 지원서를 하나 제출해 보고, 구글시트에
새 행이 추가되는지, 첨부파일 링크가 드라이브에 정상 저장되는지 확인하세요.

## 나중에 스크립트를 수정하고 싶다면

1. 시트 → 확장 프로그램 → Apps Script 에서 코드 수정
2. 저장
3. **배포 → 배포 관리 → 연필(수정) 아이콘 → 새 버전 → 배포**
   (⚠️ "새 배포"를 다시 누르면 URL이 바뀌어 버립니다. 기존 배포를 "수정"해야
   URL이 그대로 유지됩니다.)

## 데이터는 어디서 확인하나요?

- **지원자 목록**: 구글시트의 "지원자" 탭 — 제출 시각순으로 한 줄씩 쌓입니다.
- **첨부파일**: 구글드라이브의 **"라이프해킹 채용 지원서"** 폴더 — 지원자명이
  파일명 앞에 붙어 저장됩니다. 시트의 각 파일 링크 칸을 클릭하면 바로 열립니다.
